# Submissions

One folder per model, one sub-folder per run: `submissions/<model>/<run>/` holds
the **exact source the model produced** — nothing is edited by hand (see
[METHODOLOGY.md](../METHODOLOGY.md)). That tree is what the evaluator grades.

## Producing a run — `model-runner`

Runs are generated by the cross-platform **model-runner**: you name the model, it does
the rest. Each run is **two passes** in the same folder:

1. **build** — feed `PROMPT.md`; the model writes the whole project.
2. **review** — feed `PROMPT.md` + [`PROMPT-REVIEW.md`](../PROMPT-REVIEW.md); the model
   reviews its own work, runs it, validates against the brief, and applies a final patch
   (the "second chance" every model gets).

```pwsh
dotnet run --project model-runner -- sonnet-5          # next run, 2-pass, writes the sidecar
dotnet run --project model-runner -- gpt-5-5 --effort xhigh
dotnet run --project model-runner -- gemini --no-review   # single pass
dotnet run --project model-runner -- --list               # the model matrix
```

It times both passes, captures token/cost usage when the CLI reports it (Claude via
`--output-format json`), and writes the `<run>.meta.json` sidecar below automatically.

> **Auth note.** `model-runner` shells out to the model's CLI, which must be signed in.
> For `claude`, an `ANTHROPIC_API_KEY` in your environment **takes precedence** over your
> claude.ai login — if that key has no credit the run fails with *"Credit balance is too
> low"* and writes an empty run. Run with it **unset** to use your claude.ai subscription:
> `unset ANTHROPIC_API_KEY` (bash) / `$env:ANTHROPIC_API_KEY=$null` (PowerShell). This
> matters most when invoking `model-runner` from inside another Claude Code session, which
> sets that variable.

## Run provenance — `<model>/<run>.meta.json`

Alongside each run dir you can drop a **sidecar** describing *how the submission
was produced* — the harness/CLI, reasoning effort, wall-clock, tokens, cost. This
is **authored provenance**, not something the evaluator measures, so it lives
*outside* the graded tree (a sibling file, `run1.meta.json`, not a file inside
`run1/`). That keeps it away from the Roslyn / secret / PAN scans entirely.

`docs/generate-data.ps1` joins the sidecar into each run when it builds the site
data, and the site shows it in the **Provenance** section of a run's detail panel.
Every field is optional — empty or omitted fields simply don't render.

```jsonc
// submissions/opus-4-8/run1.meta.json
{
  "harness":        "Claude Code",     // agent / CLI that drove the model
  "harnessVersion": "2.x",
  "effort":         "high",            // reasoning effort setting
  "durationSec":    2600,              // wall-clock across all passes
  "passes":         2,                 // 2 = build + review; 1 = single-pass
  "attempts":       1,                 // retries of the whole run (1 = not retried)
  "tokensIn":       42000,             // optional
  "tokensOut":      88000,             // optional
  "costUsd":        3.10,              // optional
  "promptVersion":  "PROMPT.md",       // which prompt was used
  "producedAtUtc":  "2026-07-05",      // when the run was generated
  "notes":          "no hints, no rubric"
}
```

| field | type | shown as | notes |
|---|---|---|---|
| `harness` | string | Agent / CLI | joined with `harnessVersion` when both are set |
| `harnessVersion` | string | — | appended to `harness` |
| `effort` | string | Effort | e.g. `low` / `medium` / `high` |
| `durationSec` | number | Duration | total across passes, formatted `Xm Ys` / `Xh Ym` |
| `passes` | number | Passes | `2` renders as `2 (build + review)` |
| `attempts` | number | Attempts | retries of the whole run; only shown when > 1 |
| `tokensIn` / `tokensOut` | number | Tokens (in/out) | shown as `42k / 88k` |
| `costUsd` | number | Cost | shown as `$3.10` |
| `promptVersion` | string | Prompt | e.g. `PROMPT.md` or `PROMPT.md@<sha>` |
| `producedAtUtc` | string | Produced | date the run was generated |
| `notes` | string | (free text) | rendered below the rows |

Keys prefixed with `_` (e.g. `_comment`) are ignored by the renderer.

After editing a sidecar, regenerate the site data:

```pwsh
./docs/generate-data.ps1
```
