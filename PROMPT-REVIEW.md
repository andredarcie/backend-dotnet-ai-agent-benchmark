# Second pass: review your own work

You've just built this project (in this folder) to satisfy the brief above (`PROMPT.md`). Before it
ships, do a rigorous **second pass on your own work** — the way a senior engineer reviews a pull request
they're about to merge into production.

The question to answer honestly: **does this actually do what the task asked, to a production standard —
not just look like it does?** Re-read the brief and your own code with fresh, skeptical eyes, and find
where it falls short: missing or wrong behaviour, things that only appear to work, unsafe or sloppy code,
anything you'd flag in review.

**Verify it however you judge best.** You decide what's worth doing to gain real confidence — reading the
code, building it, running the tests, standing the system up and exercising it, or whatever else. Don't
assume it works; check what matters.

Then **fix everything you find.** This is your final pass: prioritise correctness and production-readiness,
keep the same scope (no new features, no gold-plating — removing accidental complexity is a valid fix),
and leave the project genuinely ready to ship.

When you're done, briefly note what you checked, what you found, and what you changed.
